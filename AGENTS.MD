# AGENTS.md - Volunteer Platform Development Guide

## Обзор проекта

Платформа для волонтёрства, объединяющая заказчиков (студентов/преподавателей, нуждающихся в помощи) и исполнителей (студентов-волонтёров). Проект состоит из двух основных частей: бэкенд (NestJS + PostgreSQL + Prisma) и фронтенд (React).

---

## Общие правила для всех AI-агентов

1. **Язык общения**: Всегда отвечайте на языке запроса пользователя. Если запрос на русском — ответ на русском, если на английском — на английском.
2. **Стиль кодинга**: Следуйте существующим конвенциям проекта:
   - Используйте NestJS-модули для бэкенда
   - Для фронтенда — React + TypeScript, следуйте существующей структуре компонентов
   - Предпочитайте Prisma ORM для работы с базой данных, используйте сырые SQL только в крайних случаях (как в методе `findAll` для задач с сложной пагинацией и фильтрацией)
3. **Ошибки компиляции**: Всегда проверяйте, что код компилируется перед запуском контейнеров. Часто встречаемые ошибки:
   - Незакрытые строки/скобки
   - Методы класса определены после их использования (TypeScript не находит их)
   - Отсутствие полей в ответах API, которые ожидает фронтенд (как в случае с полем `skills` для списка задач)
4. **Docker**: При обновлении кода всегда используйте `docker-compose up -d --build --force-recreate <service>` чтобы избежать проблем с кэшем образов.

## Бэкенд (backend/)

### Технологии

- **Фреймворк**: NestJS
- **База данных**: PostgreSQL
- **ORM**: Prisma
- **Аутентификация**: JWT
- **Контейнеризация**: Docker + Docker Compose

### Структура ключевых модулей

1. **`src/tasks/tasks.service.ts`** — основной сервис для работы с задачами:
   - Основные методы: `create`, `findAll`, `findOne`, `update`, `remove`, а также методы для смены статусов задач (`publish`, `start`, `sendToReview`, `approve`)
   - Важно: Метод `findAll` использует сырой SQL для сложной пагинации и фильтрации, обязательно включайте в запрос все поля, которые нужны фронтенду (в том числе `skills`)
   - Приватные методы: Всегда размещайте `ensureDeadlineIsValid` и `ensureSkillsExist` в начале класса, чтобы TypeScript их корректно обнаружил
2. **`src/skills/skills.service.ts`** — сервис для управления навыками
3. **`src/users/users.service.ts`** — сервис для управления пользователями
4. **`prisma/schema.prisma`** — схема базы данных Prisma

### Частые проблемы и их решения

- **Ошибка TS2339: Property 'X' does not exist on type 'Y'**: Обычно возникает, когда метод класса определён после того, как он используется. Переместите метод в начало класса.
- **Отсутствует поле в ответе API**: Проверьте сырой SQL-запрос в `findAll` — убедитесь, что все необходимые поля подключены через подзапросы (как `skills` для задач).
- **Контейнер не обновляется**: Используйте флаги `--build --force-recreate` при запуске docker-compose, чтобы пересобрать образ с нуля.

### Команды для разработки

```bash
# Сборка и запуск бэкенда
docker-compose up -d --build --force-recreate backend

# Проверка ответа API
curl -s http://localhost:3001/api/tasks | python -m json.tool
```

## Фронтенд (frontend/)

### Технологии

- **Фреймворк**: React
- **Язык**: TypeScript
- **API клиент**: Собственная обёртка над fetch в `src/lib/api.ts`

### Структура ключевых файлов

1. **`src/lib/api.ts`** — клиент для взаимодействия с бэкендом, содержит все типы для API ответов
2. **`src/app/tasks/page.tsx`** — страница со списком задач (ранее падала из-за отсутствия поля `skills` в ответе API)
3. **`src/app/tasks/[id]/page.tsx`** — страница одной задачи (всегда работала корректно, так как использует `findOne` который подключает все нужные поля через Prisma)

### Частые проблемы и их решения

- **Ошибка "Cannot read properties of undefined (reading 'map')"**: Обычно возникает, когда фронтенд пытается вызвать `.map()` на несуществующем массиве (например, `skills` не пришёл с бэкенда). Всегда добавляйте проверки на существование массивов, или гарантируйте что они всегда приходят с бэкенда (даже пустыми).
- **Проблемы с маршрутизацией**: Проверьте соответствие параметров в `app/` директории Next.js.

### Команды для разработки

```bash
# Сборка и запуск фронтенда
docker-compose up -d --build --force-recreate frontend

# Проверка работы в браузере
# Откройте http://localhost:3000
```
